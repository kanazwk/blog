hexo ブログ導入まとめ(Windows PC)

2019/12/31 カテゴリー tech

Windows PCにてhexo + GitHub pagesでブログを公開しましたので、導入方法をまとめます。

GitHubアカウントの作成

GitHubにサインアップします。アカウントを作成し、リポジトリを作りましょう。

https://github.com/

PCにGit for Windows を導入

Git(バージョン管理ソフトウェア)をインストールします。

https://gitforwindows.org/

GitHubとの連携にも使用しますし、後ほど導入するhexoの各種コマンドはGit Bashと呼ばれるターミナルで操作します。

※Macなどと違い、Windowsコマンドプロンプトはイマイチ操作しにくいので、Bashターミナルをお勧めします。もちろんWindowsのコマンドプロンプトやPowerShellターミナルでも操作可能です。

Node.jsをインストール

Node.jsのWindows版をインストールします。

http://nodejs.org/

GitHub Pagesの設定

作成したリポジトリにてGitHub pagesの設定を行います。

GitHub pagesの設定方法はこちらを参照。
https://qiita.com/kanazwk/items/947db76cfa2d879fbfe0

hexoのインストール

hexoをインストールします。

リポジトリフォルダに移動し、hexo initコマンドを実行します。
その後npm installを実行します。おそらくhexoに必要な依存関係のあるパッケージを、リポジトリフォルダ配下にインストールしているようです。

ローカルでhexoサービスを起動します。ブラウザで、http://localhost:4000/を表示すると、デフォルトテーマlandscapeのサイトが表示され、動作確認ができます。今後も多用することになるコマンドの一つです。

ブログトップページの表示が確認できたら、プロンプトでCRTL + Cを実行して終了します。

GitHub Pagesを表示する

GitHub PagesのURLをブラウザで表示し、hexo server実行時と同様の表示がされればひとまず成功です。

http://<GitHubユーザ名>.github.io/

各種テーマの導入も比較的簡単で、WordPressの導入と比較するとかなり容易に導入できるのではないでしょうか。ここまで無料であり、最低限ドメインの取得費用だけで済むかと思います。

GitHub Pagesはアクセス数に制限がありますが、Netlifyに移行することによってそれもカバーできます。
